# PM社区 UI 风格基线

## TL;DR

PM社区后续网页设计默认参考两套来源：

1. 已安装的 `ui-ux-pro-max` skill  
2. 本地参考页 `http://localhost:8888/`

目标不是直接照搬，而是提炼成一套正式社区网站长期可复用的视觉规则。

## 核心风格规则

| 维度 | 规则 |
|---|---|
| 主场景 | 深色背景为主，内容通过卡片与高光层级被托起 |
| 品牌高光 | 紫粉渐变集中用于品牌、主按钮、关键数字、重点标签，不全页滥用 |
| 内容密度 | 采用社区信息流密度，卡片结构统一为 `标签 -> 标题 -> 摘要 -> 作者/时间/互动` |
| 社区感 | 优先展示作者、时间、热度、回复、点赞、在线态等“活人痕迹” |
| 交互 | hover 上浮、边框高亮、阴影增强，动画 180ms-240ms |
| 阅读性 | 深色但不压抑，正文对比度足够，长文本区保持清晰留白 |

## 颜色方向

| Token | 建议值 |
|---|---|
| `bg-dark` | `#0f172a` |
| `bg-card` | `#1e293b` |
| `bg-hover` | `#334155` |
| `text-primary` | `#f1f5f9` |
| `text-secondary` | `#94a3b8` |
| `brand-gradient` | `linear-gradient(135deg, #6366f1 0%, #8b5cf6 100%)` |
| `accent-gradient` | `linear-gradient(135deg, #8b5cf6 0%, #ec4899 100%)` |

## 页面结构规则

| 页面类型 | 结构规则 |
|---|---|
| 首页 | Hero -> 热门话题 -> 精选文章 -> 茶水间预览 -> 工具入口 |
| 列表页 | 顶部筛选条 -> 内容流 -> 右侧辅助信息 |
| 详情页 | 主内容 -> 互动区 -> 评论 -> 作者卡 / 相关推荐 |
| 聊天页 | 频道列表 -> 消息流 -> 输入区 -> 在线信息 |
| 个人页 | 资料概览 -> 统计 -> 内容分区 |

## 禁止项

| 类型 | 规则 |
|---|---|
| 文案 | 不出现“后续/预留/扩展位/占位/首期/以后再做”等开发口径 |
| 视觉 | 不使用廉价渐变铺满全页，不混用多套卡片风格 |
| 功能 | 不保留不可用但看起来可点击的伪功能按钮 |
| 图标 | 正式站避免用 emoji 作为主要产品图标 |

## 实施说明

- 正式站优先采用“深色社区工作台”而不是普通浅色内容站。
- 若某页面不适合纯深色，可用深色主框架 + 局部亮色卡片，但不能脱离整体风格。
- 后续新增网页时，若与本基线冲突，以本基线优先。
